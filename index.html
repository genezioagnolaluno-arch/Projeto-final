Projeto Atividade 1: Análise Estrutural Computacional de Treliças Planas
▼

Enunciado Oficial:

Desenvolver um programa completo em Python focado no cálculo estrutural clássico de treliças isostáticas lineares de grande escala. O script deve receber coordenadas de nós e barras, montar dinamicamente as matrizes de equilíbrio com os cossenos diretores e realizar a resolução do sistema linear de forças através do método de eliminação de Gauss-Jordan desenvolvido manualmente de raiz (sem uso de pacotes de álgebra linear prontos de alto nível).

Código Python Implementado (Integral):

import math
import matplotlib.pyplot as plt
import numpy as np

# =========================================================
# SISTEMA DE ANÁLISE DE TRELIÇAS PLANAS ISOSTÁTICAS
# =========================================================

N = int(input('Número de nós: '))
x = []
y = []

print('\n--- Coordenadas dos Nós ---')
for i in range(N):
    xi = float(input(f'Coordenada X do nó {i}: '))
    yi = float(input(f'Coordenada Y do nó {i}: '))
    x.append(xi)
    y.append(yi)

B = int(input('\nNúmero de barras: '))
barras = []
print('\n--- Definição das Barras ---')
for i in range(B):
    no_i = int(input(f'Nó inicial da barra {i}: '))
    no_j = int(input(f'Nó final da barra {i}: '))
    barras.append((no_i, no_j))

R = int(input('\nNúmero de reações de apoio (R): '))
apoios = []
print('\n--- Definição dos Apoios ---')
for i in range(R):
    no_apoio = int(input(f'Nó do apoio {i}: '))
    direcao = input(f'Direção da restrição (X ou Y) no nó {no_apoio}: ').upper()
    apoios.append((no_apoio, direcao))

# Inicialização das matrizes do sistema linear Ax = b
total_equacoes = 2 * N
A = [[0.0] * total_equacoes for _ in range(total_equacoes)]
b = [0.0] * total_equacoes

# Aplicação de cargas externas nos nós
print('\n--- Cargas Externas ---')
for i in range(N):
    fx = float(input(f'Carga horizontal Fx no nó {i} (0 se nenhuma): '))
    fy = float(input(f'Carga vertical Fy no nó {i} (0 se nenhuma): '))
    b[2 * i] = -fx
    b[2 * i + 1] = -fy

# Montagem dos cossenos diretores nas equações de equilíbrio dos nós
for idx, (no_i, no_j) in enumerate(barras):
    dx = x[no_j] - x[no_i]
    dy = y[no_j] - y[no_i]
    L = math.sqrt(dx**2 + dy**2)
    cos_x = dx / L
    cos_y = dy / L
    
    A[2 * no_i][idx] = cos_x
    A[2 * no_i + 1][idx] = cos_y
    A[2 * no_j][idx] = -cos_x
    A[2 * no_j + 1][idx] = -cos_y

# Inclusão das reações na matriz global
col_apoio = B
for no_apoio, direcao in apoios:
    if direcao == 'X':
        A[2 * no_apoio][col_apoio] = 1.0
    elif direcao == 'Y':
        A[2 * no_apoio + 1][col_apoio] = 1.0
    col_apoio += 1

# Algoritmo de Eliminação de Gauss-Jordan desenvolvido manualmente
n = len(b)
for i in range(n):
    pivot = A[i][i]
    if pivot == 0:
        for k in range(i+1, n):
            if A[k][i] != 0:
                A[i], A[k] = A[k], A[i]
                b[i], b[k] = b[k], b[i]
                pivot = A[i][i]
                break
    if pivot == 0:
        continue
    for j in range(n):
        A[i][j] = A[i][j] / pivot
    b[i] = b[i] / pivot
    for k in range(n):
        if k != i:
            fator = A[k][i]
            for j in range(n):
                A[k][j] = A[k][j] - fator * A[i][j]
            b[k] = b[k] - fator * b[i]

print('\n================ RESULTADOS ================')
for i in range(B):
    F = b[i]
    if abs(F) < 1e-6:
        tipo = 'Nula'
    elif F > 0:
        tipo = 'Tração'
    else:
        tipo = 'Compressão'
    print(f'Barra {i}: {F:.2f} -> {tipo}')

print('\n=============== REAÇÕES ===============')
for i in range(R):
    print(f'Reação de Apoio {i}: {b[B + i]:.2f}')

Enunciado Oficial:

Redigir uma análise crítica e comparativa dos códigos e gráficos produzidos pelas LLMs (ChatGPT, Copilot e Gemini) focado no comportamento de resolução numérica de estruturas de grande escala, robustez para N=32 e conformidade com a engenharia.

Relatório Crítico e Tabela Comparativa de Desempenho:

Critério Analítico Copilot Treliças ChatGPT (Trabalho Completo) Gemini Análise Gráfica
Corretude Numérica Aproxima forças internas via deslocamentos (qualitativo). Fiel ao cálculo clássico (cossenos diretores, reações de apoio). Batem com cálculo manual. Correto para exemplo pequeno (N=3), mas não generaliza bem.
Robustez (N=4, 8, 16, 32) Aceita N arbitrário, mas pode falhar em casos muito complexos. Escala perfeitamente para N grandes, desde que as entradas estruturadas sejam fornecidas. Limitado a N=3; falha para casos maiores sem alterações diretas no código.
Qualidade Gráfica Básicos, sem legenda de apoios, mas distinguem tração/compressão por cor. Gráficos completos: apoios, cores por intensidade, espessura proporcional e legendas de fácil compreensão. Didáticos e claros, porém extremamente simplificados.
Aderência ao Prompt Implementa Gauss-Jordan de forma manual. Implementa Gauss-Jordan puramente manual sem bibliotecas prontas de álgebra linear. Usa Gauss-Jordan manual em versão excessivamente simplificada.
Conclusão do Relatório: Com base nos resultados estruturais obtidos, determinou-se que todas as LLMs possuíram resultados corretos envolvendo a física básica do problema, mas o ChatGPT se sobressaiu por apresentar resultados muito mais robustos e corretos numericamente sob malhas de grande escala. O ChatGPT cometeu erros pontuais na estruturação de loops em scripts longos, o Copilot apresentou erros na renderização tridimensional dos gráficos visuais dos resultados e o Gemini gerou um código python com lacunas internas de generalização que impediam que ele fosse aplicado diretamente a estruturas complexas N=32 sem intervenções diretas de refatoração manual do código. Portanto, o ChatGPT apresentou a melhor resposta consolidada.
Projeto Atividade 3: Gestão Computacional Hídrica de Sistemas Urbanos
▼

Enunciado Oficial:

Construir um simulador em Python que avalie o abastecimento hídrico e calcule perdas e custos associados por bairro. O programa deve processar as variáveis via objetos estruturados ou vetores e gerar um relatório automatizado em formato executável estruturado (.docx).

Código de Simulação Integrada (Base de Desenvolvimento):

import pandas as pd
import matplotlib.pyplot as plt
from docx import Document

def calcular_risco(autonomia):
    if autonomia > 3: return "Baixo"
    elif autonomia >= 1: return "Médio"
    else: return "Alto"

def analisar_bairro(nome, demanda, volume, perda, custo):
    volume_perdido = demanda * (perda / 100)
    volume_entregue = demanda - volume_perdido
    custo_diario = demanda * custo
    autonomia = volume / demanda
    risco = calcular_risco(autonomia)
    return {
        "Bairro": nome, "Demanda": demanda, "Volume Entregue": volume_entregue,
        "Volume Perdido": volume_perdido, "Custo Diário": custo_diario,
        "Autonomia": autonomia, "Risco": risco
    }

N = int(input("Quantidade de bairros: "))
dados = []
for i in range(N):
    print(f"\n--- Bairro {i+1} ---")
    nome = input("Nome: ")
    demanda = float(input("Demanda diária (m³): "))
    volume = float(input("Volume disponível (m³): "))
    perda = float(input("Perda na tubulação (%): "))
    custo = float(input("Custo bombeamento por m³: "))
    dados.append(analisar_bairro(nome, demanda, volume, perda, custo))

df = pd.DataFrame(dados)
bairro_critico = df.loc[df["Autonomia"].idxmin(), "Bairro"]

# Geração do Documento Estruturado Oficial .docx
doc = Document()
doc.add_heading('Análise do Sistema de Abastecimento de Água', level=1)
doc.add_paragraph(f"Bairro mais crítico identificado automaticamente: {bairro_critico}")
doc.save("analise_abastecimento.docx")
